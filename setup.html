🗄️ Supabase Setup (Required)

GhostMessage uses Supabase as its database. Without this step, nothing will work — no signup, no login, no messages. Follow these steps in exact order.

01

Create a Supabase Project

Go to supabase.com → Sign up free → Click "New Project" → Give it a name → Set a database password → Click "Create new project". Wait about 60 seconds for it to finish.

02

Run the Reset File First (cleans old broken data)

In Supabase → click SQL Editor in the left sidebar → click New query → paste the entire contents of supabase_reset.sql → click the green Run button.

You should see: "Success. No rows returned."
If you see errors saying things "don't exist" — that's fine, just means they were already clean.

03

Run the Main Schema File

Still in SQL Editor → click New query again (open a fresh query tab) → paste the entire contents of supabase_schema.sql → click Run.

You should see: "Success. No rows returned."

If you see a red error — screenshot it and send to Ibratech on WhatsApp: 0554196068

04

Get Your API Keys

In your Supabase project → click Settings (gear icon, bottom of left sidebar) → click API.

You need two things:
• Project URL — looks like https://abcdefgh.supabase.co
• anon public key — a long string starting with eyJ...

⚠️ Do NOT copy the service_role key — only the anon public one.

05

Paste Keys into All 7 HTML Files (fastest way)

Open the ghostmessage folder in VS Code.

Press Ctrl + Shift + H (Find & Replace across all files).

Replace: your actual anon key → click Replace All

Press Ctrl + S to save all files.

06

Change the Admin Password

The default admin password is changeme123. Change it now before you go live.

In Supabase SQL Editor → New query → paste this (change YourNewPassword):

sql
update public.admin_config
set admin_password_hash = crypt('YourNewPassword', gen_salt('bf'))
where id = 1;

You should see: "Success. 1 row affected."

07

Enable Realtime (for live message notifications)

In Supabase → click Database in the left sidebar → click Replication → find the messages table → toggle it ON.

This makes new messages appear instantly on the dashboard without refreshing. If you skip this, messages still work — you just need to refresh to see them.

08

Test Everything

Open signup.html in your browser and create a test account.

✅ Redirected to dashboard? Everything is working perfectly!
❌ See a red error box? Screenshot it and send to Ibratech: WhatsApp 0554196068

Quick summary — 3 SQL files to run in order:

1️⃣ supabase_reset.sql → wipes old broken data
2️⃣ supabase_schema.sql → builds everything fresh
3️⃣ The UPDATE password query above → secures your admin panel

Run each one in a separate New Query tab in Supabase SQL Editor. Never combine them.

Section 02

🚀 How to Launch

There are two ways to run GhostMessage locally.

01

Method 1: Direct Open (Quickest)

Simply double-click index.html in your file manager. The site opens directly in your browser. Great for testing, but some features work better with a local server.

02

Method 2: VS Code Live Server (Recommended)

Install the "Live Server" extension in VS Code. Right-click index.html → "Open with Live Server". Your site runs at http://localhost:5500

03

Method 3: Python Simple Server

Open your terminal, navigate to the project folder, and run:

bash
# Python 3
python -m http.server 8000

# Then open: http://localhost:8000
04

Method 4: Node.js HTTP Server

bash
npx serve .
# or: npm install -g serve && serve .

Section 05

🗄️ Connect a Database

Currently GhostMessage uses localStorage (browser storage). To support multiple devices and real persistence, connect Supabase — a free PostgreSQL cloud database.

01

Create a Supabase Project

Go to supabase.com → New Project. It's free. Get your Project URL and anon public key from Settings → API.

02

Create Tables

SQL
CREATE TABLE users (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  username text UNIQUE NOT NULL,
  displayname text,
  password_hash text,
  avatar_color text,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE messages (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  recipient_username text REFERENCES users(username),
  text text NOT NULL,
  is_read boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);
03

Replace localStorage Calls

Search for localStorage.setItem and localStorage.getItem throughout the codebase and replace with Supabase client calls using the @supabase/supabase-js CDN.

04

Add Supabase CDN

html
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>

<script>
  const { createClient } = supabase;
  const sb = createClient('YOUR_URL', 'YOUR_ANON_KEY');
</script>
🚨
Never expose your Supabase service_role key in frontend code. Only use the anon public key. Set up Row Level Security (RLS) policies to protect your data.

Section 07

🎨 Change Colors

All colors are stored as CSS custom properties (variables) at the top of each file. Change them once and they update everywhere on that page.

css
/* At the top of each <style> block */
:root {
  --red:    #e63946;   /* Main accent — change to your brand color */
  --green:  #2dc653;   /* Success color */
  --yellow: #ffd60a;   /* Highlight color */
  --dark:   #0a0a0f;   /* Page background */
  --dark2:  #111118;   /* Secondary background */
  --card:   #16161e;   /* Card background */
  --text:   #e8e8f0;   /* Primary text */
  --muted:  #888899;   /* Secondary text */
}
💡
You need to update the :root { } block in every HTML file since CSS and JS are embedded. Use VS Code's global find/replace to update hex codes across all files at once.
Section 08

🖼️ Edit Logo

The logo is referenced as logo.png throughout all pages. Replacing it is simple.

01

Create your new logo

Design a PNG with a transparent background. Recommended size: 512×512px minimum. Tools: Canva, Adobe Illustrator, Figma.

02

Replace the file

Name your new logo file exactly logo.png and drop it into the project root folder, replacing the existing one. Done — all pages update automatically.

03

Update favicon (optional)

The favicon is also set to logo.png via <link rel="icon" href="logo.png"/> in every page's <head>. For a proper favicon, convert to .ico format using realfavicongenerator.net.
